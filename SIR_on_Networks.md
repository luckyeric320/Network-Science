# SIR 模型、接触过程与网络渗流

## 第一部分：经典 SIR 模型（没有网络）

### 1.1 模型设定

一群人，总数 N，每个人处于三种状态之一：

- **S (Susceptible)**：健康但可能被感染
- **I (Infected)**：已感染，能传染别人
- **R (Recovered)**：已恢复，不会再被感染

两个参数：
- **β**：感染率。一个感染者单位时间内接触并传染一个特定易感者的速率
- **γ**：恢复率。一个感染者单位时间内恢复的速率（平均带病时间 = 1/γ）

### 1.2 均匀混合假设

经典 SIR 假设：**每个人和所有人等概率接触**。

这给出 ODE（用比例 s = S/N, i = I/N, r = R/N）：

```
ds/dt = -β s i
di/dt =  β s i - γ i
dr/dt =  γ i
```

初始时刻几乎所有人都是 S，即 s ≈ 1，所以 di/dt ≈ (β - γ)i。

- β > γ → i 指数增长 → **疫情爆发**
- β < γ → i 指数衰减 → **疫情消亡**

定义 **基本再生数**：

```
R₀ = β / γ
```

R₀ 就是"一个感染者在全是易感者的人群中，恢复前平均能感染几个人"。

> R₀ > 1 → 爆发；R₀ < 1 → 消亡。

### 1.3 这个模型的问题

均匀混合 = 每个人跟所有人接触，这显然不现实。现实中人只跟自己的朋友/同事/家人接触。这些接触关系构成一个**网络（图）**。

## 第二部分：把 SIR 搬到网络上

### 2.1 网络上的 SIR 规则

给定一个无向图 G = (V, E)。每个节点是一个人，每条边代表两个人之间有接触。

规则（连续时间）：
1. 感染节点对**每个**相邻的易感节点，以速率 β 尝试传染
2. 感染节点以速率 γ 自己恢复

注意和经典 SIR 的区别：感染只能沿着边传播，不是所有人对所有人。

### 2.2 一条边上的传播概率

考虑一条边，一端是感染者 A，另一端是易感者 B。

问：**A 在恢复之前，能通过这条边把 B 感染的概率是多少？**

这是两个独立的指数分布赛跑：
- A 恢复：等待时间 ~ Exp(γ)
- A 通过这条边传染 B：等待时间 ~ Exp(β)

两个独立指数分布，谁先到达？这是概率论基本结论：

```
P(传染先于恢复) = β / (β + γ)
```

> 推导：设 X ~ Exp(β), Y ~ Exp(γ) 独立。
> P(X < Y) = ∫₀^∞ β·e^{-βx} · P(Y > x) dx
>           = ∫₀^∞ β·e^{-βx} · e^{-γx} dx
>           = β ∫₀^∞ e^{-(β+γ)x} dx
>           = β / (β + γ)

我们把这个概率记作：

```
T = β / (β + γ)
```

T 叫做 **transmissibility（传播性）**。

**关键性质**：不同边上的传播是**相互独立的**。A 能不能传染 B，跟 A 能不能传染 C，是独立事件（各自概率都是 T）。

## 第三部分：SIR = 键渗流（核心等价）

### 3.1 渗流是什么

**键渗流（Bond Percolation）**：给定一个图，每条边独立地以概率 p 保留（"开"），以概率 1-p 删除（"关"）。然后看剩下的图的连通性。

核心问题：当 p 增大时，图中会突然出现一个**巨大连通分量（giant component）**——占总节点数有限比例的连通子图。这个转变发生在某个临界值 p_c。

### 3.2 等价关系

回到网络 SIR。疫情结束后，回头看每条边：
- 有的边成功传播了疾病 → "开"
- 有的边没有传播 → "关"
- 每条边独立地以概率 T = β/(β+γ) 是"开"的

所以：

> **网络上 SIR 疫情结束后被感染过的节点集合 = 键渗流中从初始感染节点出发的连通分量。**

等价对应：

| SIR 概念 | 渗流概念 |
|---------|---------|
| 传播概率 T = β/(β+γ) | 键保留概率 p |
| 零号病人 | 渗流的起始节点 |
| 最终被感染的所有人 | 起始节点所在的连通分量 |
| 疫情大规模爆发 | 出现巨大连通分量 |
| 爆发阈值 | 渗流临界值 p_c |

**为什么这个等价成立？**

想象上帝视角：在疫情开始前，先掷骰子决定每条边是否"通"（概率 T）。然后疫情从零号病人开始，沿着"通"的边传播——能到达的就是所有被感染的人。这和渗流完全一样。

严格地说，因为每条边的传播是独立的、概率是 T、而且一个节点一旦被感染恢复后就不会再被感染（不会重复经过同一条边），所以 SIR 的最终状态和键渗流的连通分量**分布完全相同**。

## 第四部分：在随机网络上求阈值

### 4.1 问题

现在我们知道 SIR ↔ 渗流，问题变成：**对于给定的网络，渗流的临界阈值 T_c 是多少？**

T > T_c → 疫情爆发；T < T_c → 疫情消亡。

### 4.2 Configuration Model（配置模型）

假设网络的度分布为 p_k（节点有 k 条边的概率），网络是随机生成的（给定度分布后随机连边）。

定义度分布的**生成函数**：

```
G₀(x) = Σ_k p_k · x^k = p₀ + p₁x + p₂x² + p₃x³ + ...
```

生成函数就是把概率分布编码成一个多项式/幂级数。它有几个好用的性质：

- G₀(1) = 1（概率归一化）
- G₀'(1) = Σ k·p_k = ⟨k⟩（平均度）
- G₀''(1) = Σ k(k-1)·p_k = ⟨k²⟩ - ⟨k⟩

### 4.3 剩余度分布

沿着一条随机的边走到一个节点，这个节点还有多少**其他**边？

这不是 p_k，因为高度节点有更多边，更容易被"走到"。一个度为 k 的节点被走到的概率正比于 k（它有 k 条边可以被选中）。走到后，它还剩下 k-1 条其他边。

所以"剩余度"的分布是：

```
q_k = (k+1) · p_{k+1} / ⟨k⟩
```

（度为 k+1 的节点，被选中概率正比于 k+1，剩余度为 k）

它的生成函数是：

```
G₁(x) = G₀'(x) / G₀'(1) = Σ_k q_k · x^k
```

### 4.4 渗流后的生成函数

每条边以概率 T 保留。一个度为 k 的节点，保留后的有效度 ~ Binomial(k, T)。

保留后的度分布生成函数：

```
G₀(x; T) = G₀(1 - T + Tx)
```

> 推导：原来度为 k 的节点，每条边独立保留概率 T。
> 有效度 j ~ Bin(k, T)，生成函数为 (1-T+Tx)^k。
> 对所有 k 求期望：Σ_k p_k (1-T+Tx)^k = G₀(1-T+Tx)。

同理：

```
G₁(x; T) = G₁(1 - T + Tx)
```

### 4.5 巨大连通分量的条件

渗流理论告诉我们：巨大连通分量存在，当且仅当从一条随机边出发，期望能到达的"下一层"节点数 > 1。

"下一层"节点数 = 剩余度的期望 = G₁'(1; T)。

```
G₁'(1; T) = T · G₁'(1)
```

而 G₁'(1) = G₀''(1) / G₀'(1) = (⟨k²⟩ - ⟨k⟩) / ⟨k⟩。

所以巨大连通分量存在的条件：

```
T · (⟨k²⟩ - ⟨k⟩) / ⟨k⟩ > 1
```

**临界阈值**：

```
┌─────────────────────────────────────┐
│                                     │
│   T_c = ⟨k⟩ / (⟨k²⟩ - ⟨k⟩)       │
│                                     │
│   等价地：T > T_c 时疫情爆发        │
│                                     │
└─────────────────────────────────────┘
```

### 4.6 两个例子

**例1：k-正则图**（每个人都有 k 个朋友）

⟨k⟩ = k，⟨k²⟩ = k²

```
T_c = k / (k² - k) = 1/(k-1)
```

比如每人有 5 个朋友，T_c = 1/4 = 25%。只要传播概率超过 25%，疫情就会爆发。

**例2：Erdős-Rényi 随机图**（平均度 z）

⟨k⟩ = z，⟨k²⟩ = z + z²（泊松分布的性质）

```
T_c = z / (z + z² - z) = 1/z
```

平均每人 10 个朋友，T_c = 10%。

## 第五部分：社交网络的特殊结构

真实社交网络不是随机图。它们有几个显著特征，每个都影响疫情传播。

### 5.1 度分布是重尾的（无标度网络）

社交网络中，大多数人朋友少，少数人（"超级传播者"）朋友极多。度分布近似幂律：

```
p_k ~ k^{-α},  通常 2 < α < 3
```

当 α ≤ 3 时，⟨k²⟩ 在网络无穷大时趋向无穷。代入公式：

```
T_c = ⟨k⟩ / (⟨k²⟩ - ⟨k⟩) → 0
```

**结论：在无标度网络上，无论传播概率多小，疫情都能爆发。没有阈值。**

原因：hub（高度节点）几乎一定会被感染，然后向大量邻居传播。哪怕 T 很小，hub 有几百上千条边，总有几条会传通。

### 5.2 聚类效应（你的朋友互相也是朋友）

社交网络有大量三角形。上面的推导假设网络是"局部树状的"（没有短环），聚类破坏了这个假设。

考虑一个三角形 A-B-C：

```
    A
   / \
  B———C
```

**树状假设下**：A 感染 B（概率 T），A 感染 C（概率 T），独立。一个感染者期望感染 2T 个人。

**有三角形时**：A 感染 B 后，B 尝试感染 C。但 A 可能已经感染了 C。B→C 这条边就**浪费了**——它没有把疫情传播给新人，只是重复感染了一个已经被感染的人。

定量地看：C 被感染的概率（至少一条路径通）：

```
P(C 被感染) = 1 - (1-T)² = 2T - T²
```

两条边的"效率"：如果没有三角形，两条边可以独立到达两个不同的人，期望感染 2T 人。有三角形时，两条边只到达一个人 C，概率 2T - T²。**传播效率降低了。**

**结论：聚类使 T_c 升高，疫情更难爆发。** 三角形造成了传播路径冗余。

### 5.3 社区结构

社交网络有社区——组内连接密集，组间连接稀疏。

```
┌─────────┐    少量桥接边    ┌─────────┐
│ 社区 A   │────────────────│ 社区 B   │
│ 内部密集  │                │ 内部密集  │
└─────────┘                └─────────┘
```

影响：
- **社区内**：传播快，迅速把社区内大部分人感染
- **社区间**：瓶颈。只有少数桥接边，疫情能否跨社区取决于这些桥接边是否"通"
- 全局爆发需要：桥接边数量 × 每条边传播概率 T 足够大

### 5.4 度相关性（同配性）

社交网络是**同配的（assortative）**：朋友多的人倾向于和朋友多的人交朋友。

- Hub 互相连接 → 形成一个"富人俱乐部"核心
- 疫情在核心中迅速传播
- 但核心到外围的连接相对少 → 限制了最终规模

与之相反，互联网（技术网络）是**异配的**：hub 连接大量低度节点 → 疫情通过 hub 扩散到更广范围。

## 第六部分：疫情最终规模

### 6.1 渗流的巨大连通分量大小

疫情的最终规模 = 渗流中巨大连通分量占总节点的比例 S。

设 u = 一个随机邻居**不在**巨大连通分量中的概率。u 满足自洽方程：

```
u = G₁(1 - T + Tu)
```

> 解释：沿一条边走到一个节点，这个节点不在巨大连通分量中，意味着它的所有其他邻居也都不在巨大连通分量中。每个其他邻居不在的概率也是 u，而其他邻居的数量服从剩余度分布。

然后：

```
S = 1 - G₀(1 - T + Tu)
```

> 解释：一个随机节点在巨大连通分量中 ↔ 它至少有一个邻居在巨大连通分量中。不在的概率 = 所有邻居都不在 = G₀(1 - T + Tu)。

### 6.2 怎么解

自洽方程 u = G₁(1 - T + Tu) 一般没有解析解，但可以：

1. **图解法**：画 y = u 和 y = G₁(1 - T + Tu)，找交点
2. **迭代法**：从 u₀ = 0 开始，反复代入 u_{n+1} = G₁(1 - T + Tu_n)，直到收敛
3. **对特殊分布可以解析求解**（比如泊松度分布 → ER 图）

**ER 图的例子**：度分布是泊松分布 Poisson(z)。

G₀(x) = G₁(x) = e^{z(x-1)}（泊松分布的生成函数）

自洽方程：u = e^{zT(u-1)}

令 S = 1 - e^{zT(S-1)}（做代换 S = 1 - u 的类似处理后）

这和 ER 图渗流的经典结果一致。

## 第七部分：总结

整个逻辑链：

```
第1步：SIR 模型放到网络上
        ↓
第2步：每条边独立传播，概率 T = β/(β+γ)
        ↓
第3步：SIR 最终状态 ↔ 键渗流（bond percolation）
        ↓
第4步：用生成函数求渗流阈值 T_c = ⟨k⟩/(⟨k²⟩-⟨k⟩)
        ↓
第5步：社交网络的结构（重尾度分布、聚类、社区、同配性）
        修正阈值和最终规模
        ↓
第6步：疫情最终规模 = 巨大连通分量大小，由自洽方程求解
```
