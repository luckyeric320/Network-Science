%% ---- 2(a) ----
\subsection{(a) Constructing a Small World Network}

\subsubsection*{Network Class}

The graph is stored as a dictionary \texttt{connections} mapping each node to its neighbor list. The five required methods:

\begin{lstlisting}[caption={HasNode and AddNode}]
def HasNode(self, node):
    return node in self.connections

def AddNode(self, node):
    if not self.HasNode(node):
        self.connections[node] = []
\end{lstlisting}

\begin{lstlisting}[caption={AddEdge}]
def AddEdge(self, node1, node2):
    self.AddNode(node1)
    self.AddNode(node2)
    if node2 not in self.connections[node1]:
        self.connections[node1].append(node2)
        self.connections[node2].append(node1)
\end{lstlisting}

\texttt{AddEdge} first ensures both nodes exist, then appends each to the other's neighbor list (undirected), skipping duplicates.

\begin{lstlisting}[caption={GetNodes and GetNeighbors}]
def GetNodes(self):
    return list(self.connections.keys())

def GetNeighbors(self, node):
    return self.connections[node][:]   # return a copy
\end{lstlisting}

\subsubsection*{Small World Construction}

The ring graph connects each node $i$ to $(i+j) \bmod L$ for $j = 1, \ldots, Z/2$. Since \texttt{AddEdge} handles both directions, iterating over positive offsets alone gives each node $Z$ neighbors. The modulo implements periodic boundary conditions.

\begin{lstlisting}[caption={MakeRingGraph}]
def MakeRingGraph(num_nodes, Z):
    g = networks.UndirectedGraph()
    for i in range(num_nodes):
        g.AddNode(i)
    for i in range(num_nodes):
        for j in range(1, Z // 2 + 1):
            neighbor = (i + j) % num_nodes
            g.AddEdge(i, neighbor)
    return g
\end{lstlisting}

Shortcuts are added by picking two distinct random nodes and calling \texttt{AddEdge} (which silently skips if the edge already exists):

\begin{lstlisting}[caption={MakeSmallWorldNetwork}]
def MakeSmallWorldNetwork(L, Z, p):
    g = MakeRingGraph(L, Z)
    num_shortcuts = int(p * Z * L / 2)
    AddRandomEdges(g, num_shortcuts)
    return g
\end{lstlisting}

\subsection{(b) Measuring Minimum Distances Between Nodes}

\subsubsection*{BFS Implementation}

\texttt{FindPathLengthsFromNode} does a breadth-first expansion in shells: start from \texttt{node} at distance 0, then repeatedly look at all unvisited neighbors of the current shell and assign them distance $\ell+1$.

\begin{lstlisting}[caption={FindPathLengthsFromNode}]
def FindPathLengthsFromNode(graph, node):
    distances = {node: 0}
    l = 0
    currentShell = [node]
    while currentShell:
        nextShell = []
        for currentNode in currentShell:
            for neighbor in graph.GetNeighbors(currentNode):
                if neighbor not in distances:
                    distances[neighbor] = l + 1
                    nextShell.append(neighbor)
        l += 1
        currentShell = nextShell
    return distances
\end{lstlisting}

\texttt{FindAllPathLengths} runs this BFS from every node and collects all pairwise distances into a dictionary. \texttt{FindAveragePathLength} simply averages those values.

\begin{lstlisting}[caption={FindAllPathLengths and FindAveragePathLength}]
def FindAllPathLengths(graph):
    allLengths = {}
    for node in graph.GetNodes():
        distances = FindPathLengthsFromNode(graph, node)
        for node2, dist in distances.items():
            if node != node2:
                allLengths[(node, node2)] = dist
    return allLengths

def FindAveragePathLength(graph):
    allLengths = FindAllPathLengths(graph)
    if len(allLengths) == 0:
        return 0
    return sum(allLengths.values()) / len(allLengths)
\end{lstlisting}

\section{Hospital Assignment (Network Flow)}

\subsection*{Setup}

There are $n$ injured people and $k$ hospitals. Person~$i$ can only go to hospitals reachable within half an hour, and each hospital should receive at most $\lceil n/k \rceil$ patients.

\subsection*{Flow Network}

Build a directed graph with source~$s$, sink~$t$, a node for each person, and a node for each hospital:

\begin{equation*}
    s \xrightarrow{\;1\;} p_i \xrightarrow{\;1\;} h_j \xrightarrow{\;\lceil n/k \rceil\;} t
\end{equation*}

Specifically:
\begin{enumerate}
    \item Add edge $s \to p_i$ with capacity 1 for every person~$i$. This forces each person to be sent to exactly one hospital.
    \item Add edge $p_i \to h_j$ with capacity 1 whenever hospital~$h_j$ is within half-hour driving distance of person~$p_i$.
    \item Add edge $h_j \to t$ with capacity $\lceil n/k \rceil$ for every hospital~$j$. This caps the load per hospital.
\end{enumerate}

Run max-flow (e.g.\ Edmonds--Karp) on this network. If the max flow equals~$n$, a balanced assignment exists and can be read off from the flow on $p_i \to h_j$ edges. If it is less than~$n$, no feasible assignment is possible.

The integer capacity theorem guarantees that the max-flow will be integral, so each person is assigned to exactly one hospital. The network has $n + k + 2$ nodes and at most $n + nk + k$ edges.

\section{Balloon Measurement (Network Flow)}

\subsection*{Setup}

There are $m$ balloons, $n$ atmospheric conditions, and a redundancy requirement~$k$. Each balloon~$i$ can measure conditions in a set~$S_i$ and can take at most 2 measurements total. We need every condition measured by at least $k$ different balloons.

\subsection*{Flow Network}

\begin{equation*}
    s \xrightarrow{\;2\;} b_i \xrightarrow{\;1\;} c_j \xrightarrow{\;k\;} t
\end{equation*}

\begin{enumerate}
    \item Edge $s \to b_i$ with capacity 2 for every balloon~$i$ (at most 2 measurements per balloon).
    \item Edge $b_i \to c_j$ with capacity 1 whenever $c_j \in S_i$ (balloon can measure that condition; capacity~1 prevents measuring the same condition twice with the same balloon).
    \item Edge $c_j \to t$ with capacity $k$ for every condition~$j$ (need $k$ measurements of each).
\end{enumerate}

Run max-flow. If it equals $n \cdot k$, every condition can be covered $k$ times; otherwise it cannot.

\subsection*{Example Verification}

Take $k=2$, $n=4$, $m=4$, with $S_1 = S_2 = \{c_1,c_2,c_3\}$ and $S_3 = S_4 = \{c_1,c_3,c_4\}$. Total supply is $4 \times 2 = 8$ and the required flow is $4 \times 2 = 8$. One feasible assignment: $b_1 \to c_1, c_2$;\; $b_2 \to c_2, c_3$;\; $b_3 \to c_3, c_4$;\; $b_4 \to c_1, c_4$. Total flow $= 8$. $\checkmark$

The network has $m + n + 2$ nodes and $m + \sum_i |S_i| + n$ edges.
